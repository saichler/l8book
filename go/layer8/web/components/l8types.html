Overview

L8Types is the foundational library for Layer 8. It defines all Protocol Buffer schemas, Go interfaces, and utilities needed to build distributed applications with service discovery, health monitoring, and secure communication. Every other Layer 8 component depends on it.

Protocol Buffer Schemas

The core type system is defined in .proto files compiled to Go via Docker-based code generation.

api.proto
Query expressions, conditions, comparators, aggregate functions, time-series points, data import/export, file upload/download, CSV export
services.proto
Service discovery, area mappings, model mappings, replication indices, transactions
health.proto
Health monitoring, statistics, pprof profiling, system status
sysconfig.proto
System configuration, VNet settings, connection parameters
notification.proto
Property change notifications, TSDB notifications, event propagation
web.proto
Web service definitions and plugin system
reflect.proto
Type reflection, nodes, decorators, table views

Virtual Networking (VNic)

The VNic interface defines six communication patterns for distributed messaging.

Unicast
Point-to-point delivery to a specific node
Multicast
Broadcast to all instances of a service
Round-robin
Distribute requests evenly across providers
Proximity
Route to the nearest service instance
Leader
Route to the elected leader instance
Local
Same-VNic messaging for co-located services

Service Discovery

Services register by area with automatic discovery, health tracking, and replication-aware routing. Service groups allow shared leader election across related services, reducing election traffic by roughly 80%.

Query System

The L8Query protobuf message supports expression-based filtering with comparators, conditions, sorting, pagination, and MapReduce flags. Aggregation queries use group_by, having, and aggregate functions (count, sum, avg, min, max).
