l8logfusion

Log Fusion

Distributed log aggregation and centralized storage.

View on GitHub All Components

Overview

L8LogFusion is a distributed log aggregation and fusion system. It collects, centralizes, and provides browsable access to logs from multiple sources in containerized environments. Logs are aggregated over a Layer 8 virtual network overlay and consolidated into a unified, queryable store.

Architecture

Three components work together:

Log Collector Agent

Scans directories recursively for .log and .err files
Tails files in real-time with rotation detection (size-based truncation check)
Buffers lines in a 30,000-entry queue with 512KB byte cap
Flushes on cooldown (2s idle), max age (30s), or buffer cap
Uses /proc filesystem analysis to detect actively-written files
Rescans for new files every 30 seconds

Log Server

Receives log batches from collectors via L8Bus unicast
Persists logs organized by source IP and filename
Thread-safe file handle cache for optimized write I/O
REST GET endpoint for directory listing and paginated file content (5KB per page)
Multi-instance merge for distributed queries

Web UI

File tree navigation across all collected sources
Paginated log viewing
Health monitoring via Probler integration
HTTPS on port 26000

Data Flow

  1. Collector opens log file, seeks to position
  2. Polls file every 1s, reads new lines into queue buffer
  3. Flushes buffer on cooldown, max age, size cap, or rotation
  4. Sends L8LogF message via L8Bus unicast to server
  5. Server writes to /data/logdb/{sourceIp}/{filename}
  6. Web UI queries server via REST GET with L8Query

Buffering Parameters

Poll interval
1s
Cooldown
2s flush after idle
Max buffer age
30s forced flush
Max buffer bytes
512KB
Queue capacity
30,000 lines

Key Features

File rotation handling -- detects truncation and seamlessly reopens
Crash recovery -- resume tailing from byte offset after restart
Configurable storage location (default /data/logdb)
Active file detection via Linux /proc filesystem
Protocol Buffers for efficient serialization
