Overview

L8 Pollaris is the polling, parsing, and populating model service for the Layer 8 data pipeline. It provides centralized configuration for how devices are polled, what data is collected, and how results are routed. Supports SNMP v2/v3, SSH, RESTCONF, NETCONF, gRPC, Kubernetes, and GraphQL.

Core Components

PollarisService
Service handling CRUD operations for polling configurations with distributed caching.
PollarisCenter
Central management hub with logging, initialization tracking, and cache synchronization.
TargetService
ORM-based persistent target management with PostgreSQL backend.
TargetCallback
Lifecycle hooks for target state changes with validation and collector routing.
TargetLinks
Abstraction layer routing targets to collectors, parsers, caches, and persistence.

Polling Configuration

An L8Pollaris configuration defines a polling strategy for a device class: vendor, series, family, software, hardware, version. Contains a map of L8Poll entries with execution order.

Each poll specifies: what to collect (what -- OID, command, or endpoint), which protocol, operation type (GET, MAP, TABLE), cadence plan, timeout, and attribute mappings with parsing rules.

Multi-model attribute mapping allows a single poll to target multiple device models simultaneously via map<string, string> PropertyId fields.

Target Management

Targets represent polled devices. Each target has a state (Down, Up, Maintenance, Offline), an inventory type (Network Device, GPU, K8s Cluster, etc.), and a map of hosts with protocol-specific connections.

Round-robin load balancing to collectors
Bulk start/stop operations by type
Automatic recovery on restart
Address validation
PostgreSQL-backed persistent storage

Collection Jobs

CJob tracks individual collection operations. Features include hash-based change detection (FNV-1a) to avoid reprocessing identical results, consecutive error counting, always-run flags, and per-job arguments for parameter substitution.
