l8ui

UI Library

Shared desktop and mobile UI component library.

View on GitHub All Components

Overview

l8ui is a configuration-driven JavaScript/CSS component library for building enterprise web applications. It provides both desktop and mobile surfaces with full feature parity. Modules supply only data (configs, enums, columns, forms) while all behavioral logic lives in shared library components.

Architecture

Desktop components use the Layer8D* prefix. Mobile components use Layer8M*. Both follow the same configuration-driven module pattern: a module is bootstrapped by a single factory call (desktop) or registry registration (mobile). All CRUD, navigation, table rendering, and form handling is handled by the library.

Module Pattern

A module consists of configuration-only files:

A config file defining services, models, and endpoints
Enum definitions with renderers
Column definitions using Layer8ColumnFactory
Form definitions using Layer8FormFactory
An init file that calls Layer8DModuleFactory.create() (~10 lines)

No behavioral code. No CRUD handlers. No navigation logic. Configuration only.

Theming

All components use --layer8d-* CSS custom properties defined in layer8d-theme.css. Dark mode is handled centrally via [data-theme="dark"] overrides. Components never define their own dark mode blocks.
