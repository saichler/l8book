Overview

L8Utils provides the shared building blocks used across all Layer 8 services. High-performance cache, priority queues, async logging, thread-safe maps, type registry, notifications, and a centralized resource container. All components are interface-driven and thread-safe.

Cache

In-memory cache with optional storage persistence. Supports CRUD, paginated queries with 30-second TTL, clone-based isolation, statistics tracking, and collection operations.

Post, Get, Put, Patch, Delete with optional notification
Dry-run Patch for optimized updates
Named stat functions with automatic totals
Query cache with configurable TTL and background cleanup

Queues

Two queue implementations optimized for different workloads.

ByteQueue
8 priority levels with O(1) bit operations. Blocking and non-blocking dequeue.
Queue
Generic queue for any type. Optional no-limit mode for unbounded workloads.

Logging

Async, queue-based logging framework with a 50,000-entry buffer. Supports file output with configurable directory, console output, or both. Five log levels: Trace, Debug, Info, Warning, Error. Includes pprof integration for CPU and heap profiling. Direct (synchronous) logger also available.

Notifications

Distributed state change notification system. Supports Add, Delete, Update, Replace, and Sync types plus TSDB notifications for time-series data. Property-level change granularity with sequence numbering and service-area routing.

Registry

Type registration and dynamic instance creation. Enum registration and management. Layer 8 core types pre-registered. Thread-safe operations via sync.RWMutex.

Design Principles

Interface-driven: all components expose interfaces from l8types
Thread-safe: sync.RWMutex, sync.Cond, or sync.Map throughout
Clone-based isolation: cache returns clones to prevent external mutation
Async processing: logger and cache use background goroutines
