Step 1

Prepare Your Computer

To get started, use a computer that can run the setup comfortably. One of these is recommended:

  • Linux — Arch, Ubuntu, or Mint recommended
  • macOS
Windows is not recommended. If needed, run Linux in a virtual machine such as VirtualBox.

This step prepares your computer with the tools and folders needed for the guided workflow.

Step 2

Run the Setup

Download and run the setup script. It installs the main tools, brings in the Layer 8 projects, and prepares your workspace for the next steps.

curl -O https://raw.githubusercontent.com/saichler/l8book/refs/heads/main/setup/setup.sh
chmod +x setup.sh
./setup.sh

Think of this as preparing your workspace so the rest of the process feels structured and repeatable.

Step 2

Your Workspace Is Ready

When setup finishes, your new project workspace will be available at:

~/proj/src/github.com/saichler/my-project

That is the folder you will use for the rest of this guided flow.

Step 3

Start Claude Code

Launch Claude from within your project directory:

claude

Follow the prompts to authenticate and connect your account.

Step 4

Create Your PRD

Layer 8 starts with intent, not code. In Claude, describe what you want to build:

Create a PRD using the Layer 8 Ecosystem Architecture
for <your idea in 2–3 sentences>

This gives Claude a clear starting point for turning your idea into a structured plan.

Step 4

Review the PRD

Validate the PRD comply with the rules:

Does the PRD comply with the global rules?

This helps you to align the plan and avoid Hallucinations

Step 4

Review the PRD

Ask Claude to surface gaps and concerns:

Identify any gaps or concerns in the PRD.

This helps you improve the plan before any implementation begins.

Step 4

Check for Reuse First

If gaps are identified, ask Claude whether existing projects already solve part of the problem:

Check whether any projects in ../ address these concerns.
This enforces reuse before creation — a core Layer 8 principle.
Step 5

Execute the PRD

Once the PRD is validated and approved:

PRD approved. Please implement it.

Claude will begin creating the system based on the plan you approved.

Step 6

Run the System Locally

After the build is complete, start the generated system on your own machine:

cd go
./run-local.sh

This starts the system in the workspace you prepared earlier.

Step 7

Prepare for Deployment

Before creating deployment images, obtain the secure base images (security + PostgreSQL). For testing with unsecure images, replace all:

  • Security base images with alpine
  • Postgres base images with saichler/unsecure-postgres:latest
  • Builder base images with saichler/builder-ns:latest
Step 7

Build the Images

Once the base images are ready, build the full set of Docker images:

go/build-all-images.sh

This prepares the system components for deployment.

Step 7

Deploy to Kubernetes

After the images are built, deploy the system to your Kubernetes cluster:

cd k8s
./deploy.sh

This is the final step in taking the system from generated code to a running deployment.

Complete
