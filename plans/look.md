# Layer 8 Website Design Guidance
## Radical Simplicity / Apple-Style Direction

---

# 1. Core Principle

The website must embody:

> Radical Simplicity. Deterministic Power. Zero Noise.

This is not a marketing site.  
This is a **proof surface for a new architecture paradigm**.

---

# 2. Design Philosophy

## Minimal — but intentional
- Every element must justify its existence
- No decorative UI
- No visual clutter

## Content-first
- Typography drives the experience
- Layout supports the message, not the other way around

## Calm and engineered
- No flashy animations
- No randomness
- Everything aligned, structured, predictable

---

# 3. Visual System

## Background
- Pure white or near-white (#FFFFFF / #FAFAFA)

## Colors
- Primary: Black (#000000)
- Secondary: Neutral gray (#666666 / #999999)
- Accent (very limited use):
    - Blue (#0070F3) OR
    - Subtle green (#00A86B)

## Rules
- No gradients
- No shadows unless extremely subtle
- No more than 2–3 colors total

---

# 4. Typography

## Fonts
- Primary: Inter
- Optional: JetBrains Mono (for code snippets only)

## Hierarchy

### Headline (Hero)
- Very large (48px–72px)
- Bold
- Tight line height

### Section Titles
- Medium-large (28px–36px)
- Bold

### Body
- 16px–18px
- High readability
- Plenty of spacing

## Style
- Short sentences
- No paragraphs longer than 3 lines
- Aggressive whitespace

---

# 5. Layout System

## Grid
- Strict vertical flow
- Centered content
- Max width: ~1100px

## Spacing
- Large vertical spacing between sections (80px–140px)
- Clean separation between ideas

## Alignment
- Left-aligned text
- No justified text
- Consistent margins everywhere

---

# 6. Homepage Structure

---

## 6.1 Hero Section (Critical)

### Headline
Build Production Systems in Days — Not Years

### Subtext
Layer 8 is an AI-native, intent-driven architecture that eliminates implicit complexity.

### CTA
- Explore the System
- See It Working

### Rules
- No images required
- No background visuals
- Let typography carry the impact

---

## 6.2 Proof Section

Show real systems:

- Probler (data center / infrastructure)
- L8ERP (ERP system)
- Layer 8 Book
- GitHub repositories

### Format
- Title
- One-line description
- “Built in X days”

### Goal
Replace claims with evidence

---

## 6.3 The Problem

Short, direct statements:

- Software systems are slow to build
- Architecture is implicit and fragile
- Complexity compounds over time

No paragraphs. Just clarity.

---

## 6.4 The Shift

Explain the Layer 8 approach:

- Intent-driven services
- Runtime API discovery
- Deterministic execution

Optional: simple diagram (no heavy visuals)

---

## 6.5 How It Works

Flow:

Service → Contract → Runtime → Exposure

Explain:
- No predefined endpoints
- APIs emerge dynamically
- Security is built-in

Keep it concise and structured.

---

## 6.6 Call to Action

Avoid:
- Contact Sales
- Book a Demo

Use:
- Explore the System
- Run It Locally
- Read the Architecture

---

# 7. Interaction Design

## Hover
- Subtle color change only

## Scroll
- Simple reveal (fade/slide)
- No complex animations

## Behavior
- Fast
- Predictable
- No surprises

---

# 8. Content Style (VERY IMPORTANT)

## Tone
- Direct
- Confident
- Technical

## Avoid
- Buzzwords (AI-powered, scalable, etc.)
- Marketing fluff
- Long explanations

## Prefer
- Short, strong statements
- Evidence over claims
- Precision over persuasion

---

# 9. What NOT to Include

- Stock photos of people
- Decorative graphics
- Overly complex diagrams
- Marketing banners
- Testimonials (initially)

---

# 10. Key Differentiator

The website should make the user think:

"Either this is wrong… or this changes everything."

That tension drives engagement.

---

# 11. Implementation Notes

## Recommended Stack

## Why
- Fast iteration
- Clean layout control
- Minimal overhead

---

# 12. Final Rule

If something feels unnecessary:

REMOVE IT.

Simplicity is not a style.  
It is the product.
